# B-plan 输出：工作项拆解与步骤队列

## 工作项（Work Items）拆解（本示例采用 1 个工作项）
为保持示例最小闭环，本轮先只做 1 个工作项：

- workItemId: **login**
- 交付内容：注册/登录/登出/受保护接口 + 最小测试 + 文档
- 验收：本地可跑通 + 失败路径覆盖 + 不泄露敏感信息

> 如果要拆成 2-3 个工作项（更贴近真实）：
> - auth-core（用户模型/密码哈希/注册）
> - auth-session（session/cookie/登录登出/保护路由）
> - auth-tests（测试与文档）
> 但此处为演示简化为 login 一个 work item。

## 步骤队列（建议）
1. login-implement（C）：实现 API 与会话逻辑
2. login-test（D）：验证/测试
3. login-accept（B）：验收与必要修正（不通过则插入 fix）
4. b-summary（B）：总结

## 验收点（Acceptance）
### login-implement
- 注册：POST /auth/register
- 登录：POST /auth/login（成功 set-cookie）
- 登出：POST /auth/logout（清 session）
- 保护接口：GET /me（需要登录）
- 密码：bcrypt 哈希存储
- 错误：统一登录失败信息

### login-test
- 成功路径：注册→登录→访问 /me→登出→/me 失败
- 失败路径：错误密码登录失败；未登录访问 /me 失败

### login-accept
- 核对非目标未引入（OAuth/MFA）
- 核对 docs/README 或 steps 输出里有运行说明与 curl 示例
