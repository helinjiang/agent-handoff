# AgentRelay 路线图

本文档定义 AgentRelay 的版本规划与里程碑。

---

## 版本概览

| 版本 | 主题 | 核心能力 | 状态 |
|------|------|----------|------|
| v0.1 | MVP | CLI 基础 + TRAE 辅助模式 | 🚧 进行中 |
| v0.2 | 增强 | 半自动执行 + 剪贴板集成 | 📋 规划中 |
| v0.3 | 自动化 | 视觉自动化 + 日志记录 | 📋 规划中 |
| v0.4 | 扩展 | 多执行端支持 | 💡 设想中 |
| v0.5 | 平台 | Web UI + 多工作空间 | 💡 设想中 |

---

## v0.1：MVP（当前版本）

### 目标

在 TRAE 多任务隔离上下文环境下，实现最小可用的产物驱动协作流程。

### 里程碑

| 里程碑 | 目标 | 验收标准 |
|--------|------|----------|
| M1 | 项目初始化 | TypeScript 项目可编译，CLI 可运行 |
| M2 | init 命令 | 能创建标准工作空间结构 |
| M3 | status 命令 | 能显示当前阶段和产物状态 |
| M4 | next 命令 | 能输出 TRAE 提示词 |
| M5 | 发布 | npm 发布，文档完善 |

### 任务拆解（Issue 列表）

#### I-001：项目初始化与基础架构
**优先级**：P0  
**描述**：初始化 TypeScript 项目，配置构建、测试、lint  
**验收标准**：
- [ ] package.json 配置完成
- [ ] tsconfig.json 配置完成
- [ ] vitest/jest 测试框架配置
- [ ] ESLint + Prettier 配置
- [ ] `npm run build` 成功
- [ ] `npm test` 可运行

---

#### I-002：CLI 框架搭建
**优先级**：P0  
**描述**：使用 commander/yargs 搭建 CLI 框架  
**验收标准**：
- [ ] `agent-relay --help` 显示帮助
- [ ] `agent-relay --version` 显示版本
- [ ] 支持 subcommand 模式
- [ ] 错误处理友好

---

#### I-003：init 命令实现
**优先级**：P0  
**描述**：实现 `agent-relay init <workspace>` 命令  
**验收标准**：
- [ ] 创建工作空间目录
- [ ] 生成 `state.json`
- [ ] 生成 `brief.md` 模板
- [ ] 目录已存在时提示错误
- [ ] 支持 `--path` 指定父目录

---

#### I-004：产物检测模块
**优先级**：P0  
**描述**：实现产物文件存在性检测  
**验收标准**：
- [ ] 能检测所有标准产物是否存在
- [ ] 能读取 `state.json` 内容
- [ ] 能推断当前阶段
- [ ] 单元测试覆盖

---

#### I-005：status 命令实现
**优先级**：P0  
**描述**：实现 `agent-relay status [workspace]` 命令  
**验收标准**：
- [ ] 显示当前阶段
- [ ] 显示各产物完成状态（✅/⬜）
- [ ] 显示建议的下一步操作
- [ ] 支持 JSON 输出格式（`--json`）

---

#### I-006：提示词模板系统
**优先级**：P0  
**描述**：实现 TRAE 提示词生成  
**验收标准**：
- [ ] 为每个阶段定义提示词模板
- [ ] 模板支持变量替换
- [ ] 提示词包含：上下文、目标、产物路径、验收标准
- [ ] 模板可配置（未来从文件加载）

---

#### I-007：next 命令实现
**优先级**：P0  
**描述**：实现 `agent-relay next [workspace]` 命令  
**验收标准**：
- [ ] 根据当前阶段输出对应提示词
- [ ] 提示词可直接复制到 TRAE
- [ ] 显示需要读取的产物列表
- [ ] 显示需要输出的产物列表

---

#### I-008：状态机实现
**优先级**：P1  
**描述**：实现工作流状态机  
**验收标准**：
- [ ] 支持阶段推进
- [ ] 支持阶段回退
- [ ] 记录状态变更历史
- [ ] 提供 `agent-relay advance` 命令

---

#### I-009：文档与示例
**优先级**：P1  
**描述**：完善文档和示例  
**验收标准**：
- [ ] README 更新使用说明
- [ ] 提供示例工作空间
- [ ] CLI 每个命令有详细帮助

---

#### I-010：发布准备
**优先级**：P1  
**描述**：准备 npm 发布  
**验收标准**：
- [ ] package.json 配置正确

## v0.2：增强版本

### 目标

提升易用性，支持半自动执行。

### 功能清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 剪贴板集成 | 自动复制提示词到剪贴板 | P0 |
| 产物校验 | 校验产物内容格式 | P0 |
| 配置文件 | 支持 `.agentrelayrc` 配置 | P1 |
| 模板定制 | 支持自定义产物模板 | P1 |
| 日志输出 | 结构化日志，支持 debug 模式 | P2 |

### 里程碑

| 里程碑 | 目标 |
|--------|------|
| M1 | 剪贴板集成完成 |
| M2 | 产物校验完成 |
| M3 | 配置系统完成 |
| M4 | 发布 v0.2.0 |

---

## v0.3：自动化版本

### 目标

支持基于视觉的 TRAE 自动化执行。

### 功能清单

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 视觉识别 | 识别 TRAE 界面元素 | P0 |
| 自动输入 | 自动输入提示词 | P0 |
| 操作录制 | 录制操作日志 | P1 |
| 回放验证 | 验证执行结果 | P1 |
| 错误恢复 | 执行失败时的恢复机制 | P2 |

### 技术选型

- Puppeteer / Playwright：浏览器自动化
- OCR：文字识别（可选）
- 图像匹配：界面元素定位

### 里程碑

| 里程碑 | 目标 |
|--------|------|
| M1 | TRAE 界面元素识别 |
| M2 | 自动输入提示词 |
| M3 | 操作日志记录 |
| M4 | 发布 v0.3.0 |

---

## v0.4：扩展版本

### 目标

支持多种执行端。

### 功能清单

| 功能 | 描述 |
|------|------|
| LLM API 执行端 | 直接调用 LLM API 执行任务 |
| Shell 执行端 | 执行 Shell 命令 |
| HTTP 执行端 | 调用 HTTP API |
| 执行端插件 | 支持自定义执行端 |

---

### 功能清单

| 功能 | 描述 |
|------|------|

| 多工作空间 | 同时管理多个项目 |

| 云存储 | 产物云端存储 |

## 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| TRAE 界面变化 | v0.3 自动化可能失效 | 使用稳定的元素选择器 |
| 视觉识别准确率 | 自动化可靠性 | 提供人工干预机制 |
| 剪贴板跨平台兼容 | v0.2 功能受限 | 使用成熟的跨平台库 |
