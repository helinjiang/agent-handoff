# AGENTS.md（AgentHandoff）

本文件用于约束 AgentHandoff 仓库内的 AI Coding 行为，确保多任务/多上下文模式下能稳定协作、可回放、可审计。

> 关键点：**不要依赖对话上下文**。所有关键输入/输出必须落到"产物（Artifacts）"。

---

## 核心原则

1. **产物驱动** - 工作流真相源来自产物（文件/记录），而不是聊天记录
2. **上下文共享 = 通过产物共享** - 任何需要共享的信息，都必须写进指定产物
3. **线性步骤链** - v0.x 只做线性步骤链，不引入 DAG 并行/分支
4. **最小共享** - 只传递必要信息：目标、约束、验收标准、变更摘要、风险点
5. **可回放** - 关键决策必须能通过产物复现

---

## 统一输出规范（强制）

每个 Step 结束时必须：

1. 更新 `steps/<nn>-<id>/output.md`
2. 追加事件到 `events.jsonl`
3. 在对话末尾输出以下尾部结构：

```
## 产物更新
- 写入/修改了哪些文件（路径 + 摘要）

## 关键决策
- 做了哪些关键选择，为什么

## 风险与待确认
- 不确定点、需要人确认的问题

## 下一步交接
- 下一 Step 要做什么
- 输入是什么
- 期望输出是什么
```

---

## 设计原则

1. **线性优先** - v0.x 只做线性步骤链，保持简单可控，避免过度抽象
2. **产物驱动** - 工作流真相源来自产物（文件/记录），而非对话历史
3. **最小共享** - 只传递必要信息：目标、约束、验收标准、变更摘要、风险点
4. **渐进增强** - 从辅助模式（生成 prompt）开始，逐步演进到半自动、全自动

## 不是什么

- **不是 DAG 调度器** - 不做复杂分支/并行调度，保持线性模型
- **不是 UI Agent 框架** - 不追求单智能体自主循环（区别于 OpenClaw 类项目）
- **不是企业平台** - 不做多租户/权限/计费等平台化能力

## 目标用户

- 使用 TRAE 或类似 AI Coding 工具的开发者
- 需要多 Agent 协作的复杂项目
- 希望工作流可追溯、可复盘的团队

## 成功指标

- 一个完整工作流结束后，通过产物即可复现所有关键决策
- 新成员可以通过 events.jsonl 快速理解项目演进历史
- 多 Agent 协作不再依赖"在同一个对话里"

---

AgentHandoff 强调**协作效率**，而非**完全自治**。
