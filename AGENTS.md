# AGENTS.md（AgentRelay）

本文件用于约束 AgentRelay 仓库内的 AI Coding 行为，确保多任务/多上下文模式下能稳定协作、可回放、可审计。

> 关键点：**不要依赖对话上下文**。所有关键输入/输出必须落到"产物（Artifacts）"。

---

## 核心原则

1. **产物驱动** - 工作流真相源来自产物（文件/记录），而不是聊天记录
2. **上下文共享 = 通过产物共享** - 任何需要共享的信息，都必须写进指定产物
3. **线性步骤链** - v0.x 只做线性步骤链，不引入 DAG 并行/分支
4. **最小共享** - 只传递必要信息：目标、约束、验收标准、变更摘要、风险点
5. **可回放** - 关键决策必须能通过产物复现

---

## 统一输出规范（强制）

每个 Step 结束时必须：

1. 更新 `steps/<nn>-<id>/output.md`
2. 追加事件到 `events.jsonl`
3. 在对话末尾输出以下尾部结构：

```
## 产物更新
- 写入/修改了哪些文件（路径 + 摘要）

## 关键决策
- 做了哪些关键选择，为什么

## 风险与待确认
- 不确定点、需要人确认的问题

## 下一步交接
- 下一 Step 要做什么、输入是什么、期望输出是什么
```

---

## 最小完成定义（DoD）

一个 Step 完成必须满足：

- [ ] 输出产物已落盘（output.md 或定义的输出）
- [ ] 输出包含：产物更新 / 关键决策 / 风险与待确认 / 下一步交接
- [ ] state.json 推进到下一步（或标记 blocked）

---

## TRAE 使用约定

- 每个 Step 对应 TRAE 中的一个 Task（独立上下文）
- Task 标题：`<nn> - <stepId> - <一句话目标>`
- 完成后把结果写回 `steps/<nn>-<stepId>/output.md`
- 遇到阻塞：在 output.md 的"风险与待确认"区明确写出
